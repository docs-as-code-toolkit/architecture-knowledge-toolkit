== Expense Entry

[plantuml,expense-entry-runtime,png]
----
@startuml
actor User
participant "Mobile App" as App
participant "Backend API" as Api
participant "Budget Domain" as Domain
database "Database" as Db
User -> App: Enter expense
App -> Api: POST /transactions
Api -> Domain: Validate and record
Domain -> Db: Store transaction
Domain --> Api: Updated budget summary
Api --> App: 201 Created
@enduml
----

== Sync Recovery

When the mobile app is offline, it stores pending expense commands locally and
replays them after connectivity returns. Conflicts are reported to the user
instead of silently overwriting server data.
