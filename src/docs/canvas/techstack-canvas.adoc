= Techstack Canvas
:id: CANVAS-TECH-001
:status: draft
:owner: project maintainers
:revdate: 2026-06-27

This canvas captures the current technical choices for the toolkit. It is a
living document and must be updated when the implementation evolves.

== Documentation

* Format: AsciiDoc.
* Diagram direction: PlantUML for architecture diagrams.
* Documentation approach: Docs-as-Code.

== Metadata

* Artifact metadata schema: YAML schema in `metamodel/artifact.schema.yaml`.
* Relationship metadata schema: YAML schema in
  `metamodel/relations.schema.yaml`.
* Metadata location for now: AsciiDoc front matter.
* Deferred metadata option: sidecar YAML files if performance or usability
  requires them.
* Artifact IDs: stable, explicit, and not reused.

== Implementation

* Current validator and generator language: Ruby.
* Current validation entry point: `scripts/validate-metamodel.rb`.
* Current tests: Ruby tests in `test/`.

== AI Assistance

* Core skills: engine-independent files under `skills/`.
* First-class version 1 skills: ADRs, quality scenarios, and risks.
* Next-priority skills: stakeholders, requirements, and components.
* Runtime integration: adapter-specific documentation under `adapters/`.
* AI output status: suggestion until reviewed and committed.

== Quality Gates

* Metadata validation must pass.
* Generated per-artifact relationship views must be deterministic.
* Generated artifact-type indexes must be deterministic.
* Generator output must be deterministic and idempotent.
* Tests must cover validator and generator behavior.
* Generated output must avoid timestamps, random values, and local paths unless
  those values are explicit inputs.

== Open Technical Decisions

* Whether to keep Ruby as the long-term implementation language.
* Whether generated documentation should be committed or produced by CI.
* Which documentation build toolchain should be the primary supported path.
