== [[testing]]Testing

The testing concept uses a small test pyramid:

[cols="1,3,2", options="header"]
|===
| Level | Focus | Traceability

| Unit tests
| Metadata parsing, relation validation, ordering, and output formatting.
| Requirements and business rules once they have IDs.

| Golden-file regression tests
| Generated relationship views, artifact-type indexes, and traceability
matrices.
| QG-001 and QG-003.

| Integration tests
| Local command behavior and CI-equivalent validation/generation paths.
| Deployment and runtime scenarios.

| Render checks
| Source fragments render cleanly in supported target formats.
| QG-004.
|===

Current evidence: Ruby validation tests exist under `test/`. Fragment render
tests are still open and should be added once fragment conventions and target
formats stabilize.
