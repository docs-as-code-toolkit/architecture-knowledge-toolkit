== [[technology-constraints]]Technology Constraints

[cols="1,3,2", options="header"]
|===
| Area | Constraint | Source input

| Documentation source
| AsciiDoc is the primary source format.
| `src/docs/canvas/techstack-canvas.adoc`

| Diagrams
| PlantUML is the diagram format for architecture documentation.
| `general-semantic-contracts.md`

| Metadata
| Artifact metadata lives in AsciiDoc front matter for now.
| `src/docs/questions-and-answers.adoc`, Q-ARCH-001.

| Generated documentation
| Generated output must not be committed and must be reproducible locally and
in CI/CD.
| `src/docs/questions-and-answers.adoc`, Q-ARCH-005.

| Validator and generator
| Ruby is the current implementation language, but not a fixed long-term
architecture decision.
| `src/docs/questions-and-answers.adoc`, Q-DEV-001.

| Publishing
| GitHub Pages with plain Asciidoctor-generated static HTML is the first
publication target.
| `src/docs/questions-and-answers.adoc`, Q-OPS-004 and Q-OPS-005.

| Runtime integration
| Engine-specific behavior belongs under `adapters/`.
| `AGENTS.md`
|===
