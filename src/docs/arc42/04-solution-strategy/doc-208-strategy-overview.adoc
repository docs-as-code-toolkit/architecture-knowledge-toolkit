== [[strategy-overview]]Strategy Overview

[cols="1,3,2", options="header"]
|===
| Approach | Description | Quality goals

| Source-first artifact model
| Reviewed AsciiDoc source and metadata are the primary architecture knowledge.
Generated output is derived and not committed as source truth.
| QG-002, QG-003

| Metadata-driven traceability
| Artifacts carry stable IDs and explicit relationship metadata. Generated
views use metadata instead of prose inference.
| QG-001, QG-003

| Layered semantic contracts
| General project rules live in `general-semantic-contracts.md`; `AGENTS.md`
adapts them for automated contributors; skills add task-specific rules.
| QG-002, QG-005

| Reusable source fragments
| Shared statements, definitions, and constraints live in reviewed fragment
files and are included with `include::[]`.
| QG-004

| Deterministic validation and generation
| Validators and generators produce stable output from stable inputs and avoid
timestamps, random values, and local paths.
| QG-001, QG-002

| Engine-independent core
| Skills and templates avoid runtime assumptions. Tool-specific details live in
`adapters/`.
| QG-005

| Walking-skeleton adoption
| The initial setup workflow is useful on its own and later slices add artifact
creation, validation, generation, and SDLC support.
| QG-002, QG-005
|===
