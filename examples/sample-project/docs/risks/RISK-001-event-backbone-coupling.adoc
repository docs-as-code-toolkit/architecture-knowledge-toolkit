= RISK-001: Event Backbone Coupling

== Risk Statement

Because multiple teams publish and consume events through a shared Kafka
backbone, teams may become coupled through event schemas, partitioning choices,
retention settings, and operational incidents, leading to slower change and
reduced delivery independence.

== Assessment

[cols="1,3", options="header"]
|===
| Field | Value

| Likelihood
| Medium

| Impact
| Medium

| Timeframe
| Medium-term

| Confidence
| Medium
|===

== Mitigation Options

* Define ownership for event schemas and compatibility rules.
* Monitor consumer lag and publish operational dashboards per event stream.
* Use topic naming and access conventions that make ownership visible.
* Review high-volume or high-criticality event streams explicitly.

== Decision or Acceptance

The risk is accepted for the sample project with mitigation. It should remain
visible while Kafka is used as the shared event backbone.
