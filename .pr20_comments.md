## Core Changes

The PR introduces three main components:

### 1. **ChapterIncludeFragmentGenerator Class** (`scripts/validate-metamodel.rb`)

A new generator class that creates include fragments for arc42 chapters. The implementation:
- Groups artifacts by chapter number
- Filters out generated files (those in `generated/` directories)
- Sorts details by artifact ID for deterministic output
- Generates AsciiDoc include directives pointing to chapter detail documents
- Writes output to `docs_dir/generated/{chapter}-includes.adoc`

**Key Design Decisions:**
- Uses regex pattern matching to identify chapter artifacts (`doc-\d{2}000-.*\.adoc`)
- Extracts chapter numbers from file paths to group related details
- Generates files in a `generated/` subdirectory to keep them as derived output
- Preserves relative path references in include directives

### 2. **Integration with MetamodelValidator** (`scripts/validate-metamodel.rb`)

The generator is instantiated and called within the validator's write flow:
```ruby
chapter_includes = ChapterIncludeFragmentGenerator.new(root: @root, docs_dir: @docs_dir)
written.concat(chapter_includes.write(metadata_artifacts))
```

This ensures include fragments are generated as part of the standard validation and generation pipeline.

### 3. **Comprehensive Test Coverage** (`test/validate_metamodel_test.rb`)

Two new tests validate:
1. **Deterministic ordering**: Verifies that multiple renders produce identical output and that includes are sorted by artifact ID (DOC-01001 before DOC-01002)
2. **Generated file exclusion**: Confirms that artifacts in `generated/` subdirectories are excluded from include generation

A private helper method `write_artifact` is added to create test artifacts with proper metadata structure.

## Possible Improvements

<!-- copilot:review-comments -->

1. **Missing null/empty check in `chapter_groups`** (`scripts/validate-metamodel.rb`, line ~795): The method calls `.fetch(chapter_number, [])` but `details_by_chapter` could theoretically have nil keys if `chapter_number_for()` returns nil. Consider an explicit guard: `details = details_by_chapter.fetch(chapter_number, []) if chapter_number`.

2. **Relative path calculation robustness** (`scripts/validate-metamodel.rb`, line ~803): The `arc42_relative_path` method catches `ArgumentError` for safety, but the subsequent logic in `chapter_artifact?` assumes a single-part filename. Consider adding a comment explaining why multi-part paths are filtered out, or consider handling nested structures more explicitly.

3. **Test helper method placement** (`test/validate_metamodel_test.rb`, line ~399): The `write_artifact` helper is private to the test class but uses hardcoded metadata fields. Consider parameterizing optional fields like `status`, `owner`, and `created` to increase test flexibility for future tests.

4. **Generator instance disposal** (`scripts/validate-metamodel.rb`, line ~945): The generator is instantiated but not explicitly closed. While Ruby's garbage collection will handle it, consider whether resources should be explicitly freed or documented (though likely not needed for this use case).
