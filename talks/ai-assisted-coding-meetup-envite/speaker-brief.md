# Speaker Brief

## Communication Job

By the end, the meetup audience should understand that AI-assisted architecture
work becomes useful for teams when the agent operates inside repository-based
contracts, skills, templates, validation, and generation instead of relying on a
single chat prompt.

## Narrative Arc

1. Start with the familiar win: AI-assisted coding is fast.
2. Introduce the failure mode: architecture artifacts become random when the
   agent has no explicit architecture environment.
3. Reframe the problem: architecture work is not code completion; it needs
   context, decisions, qualities, risks, and traceability.
4. Present the architecture-knowledge-toolkit as an AI harness.
5. Walk through an ADR flow to make the idea concrete.
6. Place the toolkit beside docs-toolbox and docToolchain.
7. Close with a grounded promise: less magic, less randomness, more reviewable
   work.

## Preferred Tone

Concrete, technical, and honest about the toolkit still being under
construction. Avoid selling the repository as a finished platform. The story is
stronger when it shows the direction and the constraints clearly.

## Phrases To Keep

- Prompting skaliert schlecht. Contracts, Skills und Docs-as-Code skalieren
  besser.
- AI wird dadurch nicht magisch richtig, aber weniger zufällig falsch.
- docToolchain publiziert. docs-toolbox baut. architecture-knowledge-toolkit
  strukturiert Architekturarbeit.
