= Demo-Skript

== Ziel

Das architecture-knowledge-toolkit als Arbeitsumgebung für AI-unterstützte
Architekturarbeit zeigen, nicht als generisches Template-Repository. Die Demo
soll deutlich machen, dass Contracts und Skills eine gewünschte Arbeitsweise
beschreiben, der Menschen und unterschiedliche AI-Agents folgen können.

== Demo-Risiken

* Nicht zu viel Zeit mit Repository-Interna verbringen.
* Die beispielhafte Architekturfrage einfach halten.
* Validierung nicht als Ersatz für menschliches Review darstellen.
* Klar sagen, dass das Toolkit eine Richtung und ein Arbeitsskelett ist, keine
  fertige Architekturplattform.
* Das persönliche AI-Lernmotiv kurz halten; es ist Kontext, nicht das zentrale
  Argument.
