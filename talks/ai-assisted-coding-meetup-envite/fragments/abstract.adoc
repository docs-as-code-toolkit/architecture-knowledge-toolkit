AI-assisted Coding ist stark darin, Code und Text schnell zu erzeugen. Aber
Architekturarbeit braucht mehr als Geschwindigkeit: Kontext, Alternativen,
Qualitätsziele, Risiken, Entscheidungen, Traceability und reviewfähige
Artefakte.

Dieser Talk zeigt, warum Architekturunterstützung für AI-Agents nicht aus immer
längeren Prompts bestehen sollte. Architekturwissen sollte stattdessen als
strukturierte Docs-as-Code-Artefakte im Repository liegen. Das
architecture-knowledge-toolkit macht daraus ein praktisches Harness: Semantic
Contracts definieren Erwartungen, Skills leiten wiederholbare Arbeit an,
Templates stabilisieren Artefakte, Validatoren erzeugen Feedback und
Generatoren halten veröffentlichte Sichten konsistent.

Die grundlegendere Idee ist: Diese Contracts und Skills beschreiben eine
Arbeitsweise, nicht nur eine Prompt-Sammlung. Sie sind Anweisungen dafür, wie
Architekturarbeit durchgeführt werden soll. Dadurch werden sie als persönliches
Nachschlagewerk, als Orientierung für Kolleg:innen und als stabiler Vertrag
beim Wechsel zwischen AI-Agents nützlich.

Das Publikum sieht in einem kurzen Repository-Walkthrough einen konkreten
ADR-orientierten Workflow: vom Contract Stack über einen ADR-Skill bis zur
Validierung als Feedbackschleife.
