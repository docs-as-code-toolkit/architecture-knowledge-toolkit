== Milestone 1: Repository Foundation

* Create initial repository structure.
* Make the initial setup workflow useful for an individual architect on its own.
* Define starter schemas for artifacts and relations.
* Provide AsciiDoc templates for ADRs, quality scenarios, and risks.
* Add engine-independent AI skills.
* Add adapter placeholders for supported AI environments.

== Milestone 2: Metamodel Validation

* Validate artifact metadata against `metamodel/artifact.schema.yaml`.
* Validate relation metadata against `metamodel/relations.schema.yaml`.
* Add sample metadata to `examples/sample-project/`.
* Enforce explicit relationships across ADRs, quality scenarios, risks, and
  related components.
* Document stable ID conventions.

== Milestone 3: Deterministic Views

* Generate traceability tables from explicit metadata.
* Generate per-artifact relationship views.
* Generate architecture knowledge indexes by artifact type first.
* Evaluate later whether indexes by status, owner, and tag add enough review
  value for version 1.
* Ensure output is deterministic and idempotent.
* Add regression tests for generator output.

== Milestone 4: Skill and Adapter Hardening

* Refine reusable skills through real project use.
* Add adapter-specific invocation examples.
* Support reproducible local, GitHub Actions, and GitLab CI workflows.
* Keep adapter behavior thin and auditable.
* Document how agent suggestions become reviewed repository changes.

== Open Questions

* Which publishing target should be supported first?
* Which implementation language should be used for future generators?
* How strict should validation be for draft artifacts?
